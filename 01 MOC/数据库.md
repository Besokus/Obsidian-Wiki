# 数据库

## 学习目标
理解数据库如何在存储、索引、事务、并发控制和恢复之间做权衡，并能从开发者角度定位性能和一致性问题。

## 核心问题
```text
1. 数据如何高效读写？
2. 并发事务如何保持正确？
3. 崩溃后如何恢复数据？
```

## 学习路径
1. 先理解 [[索引]]、[[B+Tree]]、[[缓存]]。
2. 再理解 [[事务]]、[[ACID]]、[[隔离级别]]。
3. 然后学习 [[MVCC]]、[[锁]]、[[WAL]]。
4. 最后结合慢查询、分库分表、复制和分布式事务理解工程取舍。
