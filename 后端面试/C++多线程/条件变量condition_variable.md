# 条件变量condition_variable

**是一种线程同步原语，用于让线程在某个条件不满足时进入等待状态**，直到**其他线程通知条件已经满足**再被唤醒继续执行，通常与**mutex**配合使用，解决 **线程之间的等待/通知问题**。



条件变量是一种**线程同步机制**，用于让线程在某个条件不满足时进入等待状态，直到其他线程修改条件并发送通知后再被唤醒继续执行。

在 C++ 中通常使用 `std::condition_variable`，并与 `mutex` 和 `unique_lock` 配合使用。

典型使用场景是生产者消费者模型，例如消费者在队列为空时等待，生产者生产数据后通过 `notify_one()` 唤醒消费者。



**condition_variable 为什么必须和 mutex 一起用**

condition_variable 必须和 mutex 一起使用，核心原因是**需要保证“检查条件 + 进入等待”这两个操作的原子性**。

如果没有 mutex，线程在判断条件不满足后，准备进入 wait 之前，可能另一个线程已经修改了条件并发送了通知，这样就会导致当前线程错过通知，产生“**丢失唤醒**（lost wakeup）”问题。                                           

使用 mutex 可以保证在 **wait 过程中**，线程会原子地释放锁并进入阻塞状态，从而避免这种竞态条件。

同时，mutex 也用于保护共享数据，保证多个线程访问状态时的数据一致性。
