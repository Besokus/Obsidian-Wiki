# 🌟多线程同步方式

C++中常用的线程同步方式包括**互斥锁（mutex）**、读写锁（shared_mutex）、**条件变量**（condition_variable）、**原子操作（atomic）**等。

C++ 常见线程同步方式包括：

- **`std::mutex`**：互斥锁，保护临界区，保证同一时刻只有一个线程访问共享资源；
- **`std::shared_mutex` / 读写锁**：多线程同时读取数据时共享锁（支持并发读取），写操作独占锁，提高并发性能；
- **`std::condition_variable`**：用于线程间通信，例如生产者消费者模型，线程可 `wait()` 等待条件，条件满足后被 `notify_one` 或 `notify_all` 唤醒；
- **`std::atomic`**：适合无锁原子操作，如计数器或标志位。

对共享资源我会使用锁保护，并通过 `std::lock_guard` 或 `std::unique_lock` 来管理锁的生命周期，利用 RAII 避免忘记解锁或死锁问题。

**互斥锁用于保护临界区；条件变量用于线程间事件等待与通知；原子操作在轻量场景下替代锁以提升性能。**
同步的核心目标是保证共享数据访问的有序性和一致性，避免数据竞争和死锁。

C++ 推荐使用 **RAII 封装锁（std::lock_guard / unique_lock）**。
