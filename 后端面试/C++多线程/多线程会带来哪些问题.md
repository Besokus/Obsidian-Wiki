# 多线程会带来哪些问题

- 数据竞争：**多个线程同时操作共享数据**，导致数据结果不符合预期
- **死锁（最严重）**：两个或多个线程互相持有对方需要的资源，且都不释放自己的资源
- 活锁：线程没有阻塞，但一直在重复做无效的操作

例：两个线程都试图释放自己的锁来避免死锁，但每次释放后又立即尝试获取对方的锁，陷入循环

解决：引入随机等待时间、设置重试次数上限

- 线程饥饿：某些线程长期得不到 CPU 执行时间或资源
- **上下文切换开销**：当线程数量过多时，CPU大部分时间花在线程切换上，而非处理业务逻辑

解决方法：使用线程池、控制线程数、减少锁竞争
