# 🌟线程死锁的原因，如何避免死锁

线程死锁是指两个或多个线程**互相等待对方持有的资源**，**导致都无法继续执行**

**死锁四条件**：

 1️⃣ **互斥条件**：线程对资源独占访问，其他线程必须等待。
 2️⃣ **请求与保持**：线程在持有一个资源的同时申请其他资源。
 3️⃣ **不可剥夺**：资源不能被强制回收，只能由线程释放。
 4️⃣ **循环等待**

避免死锁：破坏死锁四个必要条件，C++11 的 `std::lock()` 可以同时锁定多个 mutex，避免顺序问题。

RAII + `std::lock_guard` / `std::unique_lock` 能减少因异常导致的死锁。
