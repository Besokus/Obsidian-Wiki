# 原子操作atomic和mutex的使用场景和区别

atomic 解决的是“单个变量的并发一致性”，
mutex 解决的是“一段代码/多个操作的原子性”。

| 对比维度   | atomic                      | mutex                      |
| ---------- | --------------------------- | -------------------------- |
| 保护对象   | 单个变量                    | 一段临界区（可包含多变量） |
| 原子性范围 | 单条操作                    | 多条语句                   |
| 是否阻塞   | 不阻塞（CPU 原子指令/自旋） | 会阻塞（线程睡眠/唤醒）    |
| 表达能力   | 弱（只能简单操作）          | 强（可表达复杂逻辑）       |
| 性能特征   | 低竞争快，高竞争抖动        | 稳定但有上下文切换         |
| 可维护性   | 低，易出错                  | 高，语义清晰               |
