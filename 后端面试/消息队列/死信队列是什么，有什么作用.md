# 死信队列是什么，有什么作用

死信队列（DLQ）是用于存放“无法被正常消费的消息”的专用队列。



**死信队列作用**：

-  防止消息无限重试：造成消费线程被占满、消息积压、系统雪崩（**隔离异常消息，保护主链路**）
-  提供问题排查入口：错误消息被完整保留，可用于分析异常问题
-  支持补偿与人工干预：解决bug后可将死信队列中的消息重新投递



**什么情况下消息会进入死信队列（重点）**

**常见触发条件**

1. **消费失败超过最大重试次数**
2. **消息过期（TTL 到期）**
3. **队列长度达到上限**
4. **消费者显式拒绝且不重回队列**



**总结**：死信队列用于存放在正常消费流程中无法被成功处理的消息，避免异常消息阻塞主消费链路。消息在超过最大重试次数、过期、被拒绝等情况下进入死信队列，供后续排查、补偿或人工处理。通过死信队列机制，系统能够在保证稳定性的同时提高消息处理的可靠性和可维护性。
