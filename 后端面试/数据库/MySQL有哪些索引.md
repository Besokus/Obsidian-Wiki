# 🌟MySQL有哪些索引

从数据结构看，MySQL的索引主要有以下几种:
1.**B+-Tree索引**:也就是我们常说的索引，用于等值和范围查询，左侧前缀查找;
2.**哈希索引**:只用于等值查询，不能用于排序和部分查找;
3.**空间数据索引**:主要用于空间数据查询，比如地理数据查找;
4.**全文索引**:用于全文搜索，倒排的方式
底层数据结构:

1.B+ Tree索引: 采用B+树数据结构;

2.哈希索引: 采用哈希表;

3.空间数据索引: 采用R-Tree;

4.全文索引：采用倒排索引



从功能上看，常见的有**主键索引、唯一索引、普通索引和组合索引**。在 InnoDB 中主键索引属于聚簇索引，数据和索引存储在一起，而普通索引属于二级索引，查询时通常需要回表。组合索引则需要遵循最左匹配原则。



按“存储结构”分类：聚簇索引（数据和索引在一起），二级索引（叶子节点存主键值，
需要回表）



MySQL 的 InnoDB 默认使用 **B+Tree 索引**。

**B+Tree 特点：**

- **多路平衡搜索树**，每个节点可存放多个 key。
- **非叶子节点只存索引键，不存数据；**
- **叶子节点存放所有数据，并通过链表相连。**（范围查询和排序效率高）
- 数据**只存在叶子节点**中，查找路径更短，**范围查询更高效**。



**优点：**

- IO次数少（树高低，通常 2~4 层即可存百万行数据）；
- 叶节点顺序链表可支持**范围查询**；
- 支持磁盘块按页加载（局部性原理）。



**为什么索引用B+树不用B树**

回答B+树的**特点和优势**，B+ 树叶子节点链表是 **覆盖索引 + 范围查询优化的关键**；
