# 🌟SQL 优化（查询慢怎么办）

查询慢可能由三方面原因：**SQL 语句本身**、**表结构/索引**、**系统资源**。

使用 **EXPLAIN** 分析执行计划，确认是哪条语句存在性能问题

**优化 SQL 语句**：避免 SELECT *，索引列写在 WHERE/ORDER BY 中，拆分复杂查询；

SQL语句优化方案：

- 使用索引：确保涉及的列都有适当的索引

- 避免模糊查询：%开头的通配符会使索引失效，尽量避免在查询条件中使用以%开头的LIKE语句。

  ```mysql
  SELECT * FROM table WHERE column LIKE 'value%';
  ```

- 避免使用select \*

- 通过EXPLAIN检查查询语句**是否使用了合适的索引**（索引是否失效，查看实际走的索引和理论是否一致）

**优化表结构**：建立合适索引、使用覆盖索引、考虑分表分库；

**系统层面**：增加缓存（Redis）、调整 InnoDB 配置、减少锁竞争。 

**如果追问**

“如果使用 Redis 缓存，如何保证缓存与数据库一致？” → 可以说 **先更新 DB，再删除缓存 / 延时双删 / 异步消息队列同步**。

“慢查询如何监控？” → 使用 **慢查询日志**
