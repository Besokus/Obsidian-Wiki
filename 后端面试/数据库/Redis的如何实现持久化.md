# Redis的如何实现持久化

Redis 提供了两种主要的持久化机制：**RDB 快照**和 **AOF 日志**。

**RDB（Redis Database Snapshot）** 是通过**定期将内存中的数据快照保存到磁盘**，比如生成一个 `dump.rdb` 文件。
它的优点是**恢复速度快、文件体积小**，但缺点是**可能丢失最近一次快照之后的数据**。

**AOF（Append Only File）** 则是把**每一次写操作都追加到日志文件中**，比如 `appendonly.aof`。
它的优点是**数据可靠性高**，可以通过重放命令恢复数据；**缺点是文件体积大，写入性能稍低**。

实际生产环境通常会**同时开启 RDB 和 AOF**：

- 这样在 Redis 宕机后可以优先使用 AOF 恢复数据（更完整），
- 同时 RDB 作为定期备份，提高恢复效率。

AOF 还可以通过 `appendfsync everysec` 配置为**每秒刷盘一次**，在性能和安全之间取得平衡。

我了解这些机制的主要目的是，在 Redis 用作缓存或持久层时，能根据场景选择合适的策略，
比如在高并发场景使用 AOF + 异步刷盘保证可靠性，在非关键缓存场景使用 RDB 降低磁盘负载。
