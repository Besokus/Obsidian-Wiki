# 🌟索引失效

索引会失效的场景包括：
1.使用  !=  或 <> 操作符;
2.对字段进行**运算或函数**操作;
3.使用 LIKE'%XXX'这样以**通配符开头的模糊查询;**
4.**数据类型不一致**，比如用字符串类型的字段去匹配数字类型的索引;
5.在**WHERE子句中使用 OR**，除非OR条件中的每个列都进行了索引（OR 本身不会导致索引必然失效，只要各条件都用索引，MySQL 可能使用 Index Merge；但最终是否使用索引**取决于优化器的成本评估**。）。

6. 违背“最左匹配原则”：如果创建了联合索引 `(a, b, c)`，但在查询时跳过了前面的列（a），则无法利用索引的有序性。
