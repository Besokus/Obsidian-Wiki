# 🌟MySQL 的 三大日志（Binlog Undolog Redolog）？作用是什么

Binlog 是 MySQL 的**二进制日志**，记录了所有会**修改数据库的操作**（INSERT、UPDATE、DELETE、DDL），不记录 SELECT。
主要作用有三点：

1. **主从复制**：主库将 Binlog 发给从库，实现数据同步；
2. **数据恢复**：通过重放 Binlog **可以恢复误操作数据**；
3. **审计**：可分析历史操作。

**总结一句话：**
 “Binlog 是逻辑日志，保证**主从复制和数据可恢复性**，是数据库高可用和审计的重要基础。”



**Undo log **（回滚日志）：用于**事务回滚和 MVCC**，记录的是**数据被修改前的版本**。

**Undo log是逻辑日志**，可以认为当 delete 一条记录时，undo log中会记录一条对应的insert记录，反之亦然。

当执行 **rollback** 时，就可以从 undo log 中的逻辑记录读取到相应的内容并进行回滚。



**Redo log**（重做日志）：用于崩溃恢复，保证事务的持久性。

当事务提交之后会把**所有修改信息**都存到该日志文件中,用于在刷新脏页到磁盘，发生错误时，**进行数据恢复使用**。
