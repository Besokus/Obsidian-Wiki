# 🌟MySQL主键（聚簇）索引和唯一（辅助）索引的区别

主键索引和唯一索引都能保证列唯一性，但有两个主要区别：
 1️⃣ 主键索引**不允许 NULL**，每张表只能有一个，并且在 InnoDB 中是**聚簇索引**（采用B+树索引实现），数据存储在主键索引的叶子节点中，**叶子节点存放整行数据**；
 2️⃣ 唯一索引可以有多个但要求唯一，**允许 NULL**，是二级索引，**叶子节点保存主键值和索引列**，需要**回表**查数据。
 主键更偏向于**表的唯一标识**，而唯一索引偏向**业务约束字段**。

通过**主键查询**时，只需一次 B+Tree 查找，走聚簇索引；

通过**非主键查询**时，需要先查辅助索引找到主键索引，再根据主键去**聚簇索引**中查找（“**回表**”）。



**有一种方式可以不需要回表**：覆盖索引

```mysql
SELECT email FROM user WHERE email = 'a@b.com';
```

- email 是辅助索引
- 查询的列就在索引里



由于 InnoDB 的聚簇索引叶子节点存放**整行数据**，因此辅助索引体积更小，但需要额外一次回表查询，这也是索引设计中需要权衡的地方
