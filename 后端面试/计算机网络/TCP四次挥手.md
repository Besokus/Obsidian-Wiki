# 🌟TCP四次挥手

TCP 断开连接使用四次挥手。
 第一次，主动方发送 FIN 表示**不再发送数据**（客户端 -> 服务端）；
 第二次，被动方回复 ACK 确认（服务端 -> 客户端）；
 第三次，被动方也发送 FIN 表示**它也要关闭连接**（服务端 -> 客户端）；
 第四次，主动方再回复 ACK（客户端 -> 服务端），之后进入 TIME_WAIT 状态等待 2MSL，**确保对方收到 ACK** 并防止旧连接报文影响新连接。
由于 TCP 是**全双工**的，**所以两端的关闭需要分别确认**，因此是四次挥手而不是三次。
