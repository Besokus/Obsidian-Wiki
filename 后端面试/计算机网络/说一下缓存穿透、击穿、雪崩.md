# 🌟说一下缓存穿透、击穿、雪崩

**缓存穿透**：缓存和数据库都没有 → 用布隆过滤器或缓存空值；

**缓存击穿**：热点数据失效瞬间大量请求打 DB → 互斥锁或热点永久缓存；

**缓存雪崩**：大量缓存同时失效 → 过期时间随机化 + 降级 + 本地/二级缓存。



1️⃣ **缓存穿透（Cache Penetration）**

**定义**：请求的数据 **在缓存和数据库中都不存在**，每次请求都会穿透缓存直接查询数据库。
**场景**：恶意请求不存在的用户ID或商品ID。
 **解决方法**：

- **布隆过滤器**（Bloom Filter）拦截无效请求
- **缓存空值**：对不存在的数据也写入 Redis（设置短过期时间），防止每次都访问数据库。

------

2️⃣ **缓存击穿（Cache Breakdown / Hot Key）**

**定义**：某个 **热点数据在缓存失效** 的瞬间，有大量并发请求直接打到数据库。
**场景**：高并发热门商品或秒杀活动。
**解决方法**：

- 设置热点 key 永不过期
- **互斥锁 + 单线程刷新**：只有一个线程去数据库加载数据，其他线程等待或返回旧值。
- **异步刷新缓存**：提前在**后台更新缓存**，防止瞬时失效。

------

3️⃣ **缓存雪崩（Cache Avalanche）**

**定义**：**大量**缓存**同时过期或缓存服务宕机**，短时间内大量请求直接打到数据库，造成数据库压力过大。
**场景**：所有缓存设置相同过期时间或 Redis 集群宕机。
**解决方法**：

- 缓存过期时间随机化（打散失效时间）
- 加本地缓存或二级缓存
- 增加降级方案和限流保护
