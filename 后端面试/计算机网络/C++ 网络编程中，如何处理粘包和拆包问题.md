# C++ 网络编程中，如何处理粘包和拆包问题

TCP链接是流式协议，没有消息边界

**粘包（Stick Packet）**：多个消息粘在一起，接收方一次 `recv()` 收到多个发送方的数据包。（nagle算法将小包合并在一起）
**拆包（Half Packet）**：一个完整的消息被拆成多段，接收方一次 `recv()` 只收到一部分数据。（消息太大超过最大报文长度MMS）

对于粘包和拆包问题，常见的解决方案有四种：

- 发送端将每个包都**封装成固定的长度**，比如100字节大小。如果不足100字节可通过补0或空等进行填充到指定长度；
- 发送端在每个包的末尾**使用固定的分隔符**区分消息边界，例如\r\n。如果发生拆包需等待多个包发送过来之后再找到其中的\r\n进行合并；例如，FTP协议；
- 将消息分为**头部和消息体**，头部中保存**整个消息的长度**，只有读取到足够长度的消息之后才算是读到了一个完整的消息；（粘包）
- 通过自定义协议进行粘包和拆包的处理。
- 关闭TCP的 nagle算法 （粘包）
