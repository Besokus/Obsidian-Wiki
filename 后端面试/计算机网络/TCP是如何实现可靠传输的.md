# 🌟TCP是如何实现可靠传输的

TCP 通过序列号（保证有序、不重复）、ACK 确认（保证数据被正确接收）、**超时重传、滑动窗口、流量控制**（防止**接收端缓存溢出**）和**拥塞控制**（**防止网络过载**）来实现可靠传输。

**确认应答（ACK）**

> 接收方在收到数据后，会返回 ACK 确认报文，告知发送方哪些数据已经成功接收，从而实现可靠确认。



**超时重传（Retransmission）**

> 如果发送方在一定时间内没有收到 ACK，就会触发超时重传机制，重新发送数据，保证数据最终能够送达。



**快速重传**

> 当发送方连续收到多个相同的 ACK（通常是 3 个重复 ACK），会认为某个数据段丢失，从而提前进行重传，而不必等待超时，提高传输效率。



**流量控制（Flow Control）**

> TCP 使用滑动窗口机制，由接收方通过窗口大小（rwnd）告知发送方当前还能接收多少数据，从而避免发送方发送过快导致接收方处理不过来。



**拥塞控制：**

四个核心算法：

- **慢启动**（逐步探测带宽）；
- **拥塞避免**（到达慢开始门限，线性增加）；
- **快速重传**：发送方连续收到 3 个重复 ACK则立即重传
- **快速恢复**：将拥塞窗口减半并进入**拥塞避免阶段**，避免重新慢启动

**拥塞控制：根据网络的实际承载能力动态调整发送速率，避免网络过载。**
