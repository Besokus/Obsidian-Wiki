# **怎么解决哈希冲突**

**链地址法（Separate Chaining）：最常用，HashMap/Redis 首选**

通过key找到对应的位置，当冲突发生时，将冲突的键值对直接插入对应桶的链表 / 红黑树尾部；

**优点**

- 负载因子可较大（如 `loadFactor=0.75`），哈希表利用率高；
- 链表 / 红黑树的动态扩容特性，可容纳任意多的冲突键值对；



**开放地址法（Open Addressing）：无额外空间开销，适合小容量场景**

当冲突发生时（目标地址已被占用），通过 **特定探测规则** 寻找哈希表中「空闲的地址」存储冲突键值对；（常见的就是按照顺序往后寻找，是个循环数组）

**优点**

- 数据存储连续，缓存命中率高（CPU 预读效率高）；
- 查找速度快（无链表遍历，仅需计算探测地址）。
