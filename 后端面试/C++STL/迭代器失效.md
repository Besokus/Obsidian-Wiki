# 迭代器失效

- 迭代器失效是指容器被修改后，**旧的迭代器/指针/引用不再有效**。
- 连续存储（`vector/string`）在 **扩容或中间插入/删除** 会使大量迭代器失效； 节点式容器（`list/map/set`）**插入一般不失效**，`erase` 只使被删元素失效； 哈希容器在 **rehash** 时会使 **全部** 迭代器失效。
- 规避方法：**用 `erase` 返回值继续遍历**、提前`reserve` 预容量（vector）、修改后**重新获取迭代器**、必要时选用 **迭代器稳定的容器（如list/map）**。

```c++
std::vector<int> v{1,2,3,4};
for (auto it = v.begin(); it != v.end(); ) {
    if (*it % 2 == 0)
        it = v.erase(it);   // 使用 erase 的返回值继续遍历（正确）
    else
        ++it;               // 不删除时正常前进
}
```

| 容器类型        | 底层结构 | 失效场景                                                     | 不失效场景                                                   |
| --------------- | -------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| `vector`        | 连续数组 | 1. push_back/insert 触发扩容（所有迭代器失效）；2. erase 删除元素（当前 + 后续迭代器失效）；3. resize/reserve 扩容（所有迭代器失效） | 1. 只读操作（遍历、[] 访问）；2. 尾部插入且未扩容（仅尾后迭代器失效） |
| `list`          | 双向链表 | 仅被 erase 删除的那个迭代器失效（其他迭代器均有效）          | 插入 / 删除非当前迭代器指向的元素                            |
| `map`/`set`     | 红黑树   | 仅被 erase 删除的迭代器失效（其他迭代器有效）                | 插入 / 删除非当前迭代器指向的元素                            |
| `unordered_map` | 哈希表   | 1. rehash 扩容（所有迭代器失效）；2. erase 删除的迭代器失效  | 插入且未触发 rehash（仅被插入位置附近迭代器可能失效）        |
