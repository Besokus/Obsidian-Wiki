# 🌟说一下 `mutex` 和 `condition_variable` 的使用场景

mutex 主要用于**保护共享资源**，避免多个线程同时访问导致数据竞争问题，从而保证数据一致性。

但是 mutex 只能做到“互斥访问”，如果资源暂时不可用，线程只能不断尝试加锁，这会导致忙等待（busy waiting），浪费 CPU。

**condition_variable** 的作用是让线程在条件不满足时进入阻塞状态（sleep），并在条件满足时被其他线程唤醒，从而避免无意义的 CPU 消耗。

它通常和 mutex 配合使用，**通过 wait() 释放锁**并进入阻塞状态，通过 notify_one() 或 notify_all() 唤醒等待线程。



**生产者消费者（必须会说）**

你可以补一句：

> 在生产者消费者模型中：
>
> - 当队列为空时，消费者线程通过 wait() 进入等待状态
> - 当生产者生产数据后，通过 notify_one 唤醒消费者
> - 这样可以避免消费者线程一直轮询队列，提高效率
