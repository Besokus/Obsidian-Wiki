# 动态链接和静态链接是什么？各自优缺点？

静态链接是指在**编译阶段**，编译器会把程序用到的库代码直接打包进可执行文件里；
动态链接则是在**运行时**再去加载外部的共享库，比如 `.so` 或 `.dll` 文件。

静态链接的优点是**运行时不依赖外部库，启动快**、发布方便，但缺点是可执行文件体积大、**更新库需要重新编译**。
动态链接的优点是**多个程序可以共享同一个库，节省内存，更新方便**；缺点是**启动稍慢，而且依赖库版本一致**，否则会运行失败。 

总体来说，静态链接更适合独立部署的程序，动态链接更适合系统级或大型应用的模块化场景。
