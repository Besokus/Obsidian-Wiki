# 右值引用是什么，为什么要引入右值引用？

| 类型 | 核心特征                          | 例子                                                         |
| ---- | --------------------------------- | ------------------------------------------------------------ |
| 左值 | 有名字、能取地址、生命周期持久    | `int a = 10;` 中的 `a`；`std::string s = "hi";` 中的 `s`     |
| 右值 | 无名字、不能取地址、临时 / 一次性 | 字面量 `10`、`"hi"`；表达式 `a+b`；函数返回的临时对象 `func()` |

右值引用是「专门引用右值的引用类型」，语法是 `T&&`（区别于左值引用 `T&`），核心特点：

1. 只能绑定到**右值**（临时对象、字面量），不能绑定到左值；
2. 绑定后，能接管右值的资源（因为右值即将销毁，无其他使用者）。

C++11 引入右值引用的**唯一核心目的**：避免对「临时对象」的**无意义拷贝，提升性能** —— 这在处理**大对象**（如字符串、容器、自定义类）时效果极其显著。通过**移动构造函数和移动赋值运算符**，可以**转移资源所有权**而不是进行深拷贝，从而显著提升性能，尤其在容器扩容或函数返回大对象时效果明显。



**移动后原对象是否还存在？**

移动语义不会销毁原对象，只是把原对象内部的资源 **转移（transfer）** 给新对象。被移动的对象处于 **valid but unspecified state**（有效但未指定状态）



**完美转发**

完美转发则是在模板函数中，把参数原样转发给其他函数时，保留它的左值或右值属性，避免右值信息丢失。
