# 🌟C++ 面向对象三大特性

**封装 (Encapsulation)**

- **核心：** **隐藏实现细节，提供公共接口。**
- **解释：** 封装是将数据（属性）和操作数据的方法（行为）捆绑在一起，形成一个类。我们使用 `private` 或 `protected` 访问修饰符来保护数据不被外部直接访问，只通过 `public` 接口进行受控的操作。
- **作用：** 保证数据的**安全性和一致性**。



 **继承 (Inheritance)**

- **核心：** **代码重用，建立 "is-a" 关系。**
- **解释：** 继承允许新类（派生类/子类）获取现有类（基类/父类）的属性和方法，支持多继承，是 C++ 的重要特性。
- **作用：** 极大地提高了代码的**可重用性**，并允许我们建立清晰、可扩展的类层次结构。



**多态 (Polymorphism)**

- **核心：** **一个接口，多种形态（运行时行为不同）。同一接口在不同对象上表现出不同的行为。**
- **实现方式：**
  - **动态多态 (Runtime)：** 这是 C++ 多态的核心。它通过在基类中声明 **虚函数（`virtual`）**，并使用**基类指针或引用**来调用函数，从而在**运行时**根据实际对象类型决定调用哪个函数版本。
  - **静态多态 (Compile Time)：** 通过函数重载和模板实现，在编译阶段就确定函数的调用。
- **作用：** 增强了代码的**灵活性和可扩展性**，使程序能够统一处理不同类型的对象。
