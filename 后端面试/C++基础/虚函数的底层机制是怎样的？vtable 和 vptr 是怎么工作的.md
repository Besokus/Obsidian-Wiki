# 🌟虚函数的底层机制是怎样的？vtable 和 vptr 是怎么工作的？

虚函数表（vtable）： **每个含虚函数的类在编译时**都会生成**一个vtable，里面存的是该类的虚函数地址列表**。（属于类）

虚指针（vptr）： **每个对象内部都含一个隐藏的指针（vptr），指向它所属类的vtable**。（属于对象）

动态绑定过程： 如：当用Animal* a = new Dog()时： 创建Dog对象时，会设置a->vptr指向Dog类的vtable。

当调用a->speak()，程序会通过vptr找到Dog类的vtable，调用里边的speak()地址。

- 拓展：

vtable 是**类级别**的（多个对象共享），vptr是**对象级别**的

vptr 存储在**对象内部**，而 vtable 通常存储在程序的**只读数据区**，并由**同一类的所有对象共享**。调用虚函数时，程序会通过对象中的 vptr 找到 vtable，然后查表调用对应的**函数地址**，从而实现**运行时多态**。

| 组件               | 归属     | 作用                                                         |
| ------------------ | -------- | ------------------------------------------------------------ |
| 虚函数表（vtable） | 类       | 存储该类所有虚函数的地址，程序启动时就已创建，全局唯一       |
| 虚表指针（vptr）   | 每个对象 | 指向所属类的虚函数表，是对象内存布局中额外占用的 4/8 字节（32/64 位系统） |
