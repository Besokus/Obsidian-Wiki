# 🌟C++ 智能指针（RAII）

unique_ptr：独占智能指针，独占对象所有权，同一时间只能有一个指针指向一个对象，适合独占资源的场景；**禁止拷贝构造和拷贝赋值，支持移动语义**。



share_ptr：一个**共享所有权**的智能指针，**允许对象之间进行复制或者赋值**，展示出来的就是**值语义**。

**share_ptr的引用计数机制**

引用计数并 **不存储在对象内部**，而是存储在一个 **控制块**，多个share_ptr共享同一个控制块

一个 `shared_ptr` 本质包含两个指针：

- 指针 1：指向**实际的资源**（比如 `new int(10)` 分配的内存）。

- 指针 2：指向**控制块**（control block）—— 这里存放着：

  ✅ 引用计数（ref_count）：记录有多少个 shared_ptr指向该资源；

  ✅ 弱引用计数：记录有多少个 weak_ptr指向该资源；

  ✅ 资源的析构函数 / 删除器；

当强引用计数变为 0 时，对象会被释放；而当弱引用计数也为 0 时，控制块才会被销毁。

此外使用 `make_shared` 可以将对象和控制块一起分配，从而减少内存分配次数



weak_ptr 的主要作用是**解决 shared_ptr 的循环引用**问题。

在使用 shared_ptr 时，如果两个对象互相持有对方的 shared_ptr，比如 A 持有 B，B 也持有 A，那么它们的引用计数永远不会变成 0，导致内存无法释放，这就是循环引用问题。

weak_ptr 本身不会增加引用计数，它只是“弱引用”这个对象，因此不会影响对象的生命周期。

当需要访问对象时，可以通过 **lock()** 将 weak_ptr 转换为 **shared_ptr**，如果对象还存在则访问，否则返回空。

通过将其中一方的 shared_ptr 改为 weak_ptr，就可以打破引用环，从而避免内存泄漏。



- 什么是RAII机制？ RAII 全称： Resource Acquisition Is Initialization（资源获取即初始化）

即：当你用一个智能指针去**创建**对象时，它立刻就会接管资源（如内存）； 当智能指针生命周期结束（如**离开作用域、函数返回**等），它会**自动调用析构函数**，把**资源安全释放掉**，**不需要你手动 delete**。

**RAII 核心**：将资源绑定到对象生命周期，构造获取资源，析构释放资源，利用 C++ 析构函数的自动调用保证资源不泄漏。

**解决的问题**：

- 避免资源泄漏（内存、文件、锁等），尤其是异常场景下的泄漏。
- 减少手动释放资源的冗余代码，提升代码可读性和健壮性。

通俗的讲就是，RAII就是“我拿到资源我就负责到底，我走了我就顺手把它销毁”
