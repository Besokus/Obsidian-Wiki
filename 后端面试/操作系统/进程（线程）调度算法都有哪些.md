# 进程（线程）调度算法都有哪些

1. **先来先服务 (FCFS)**，这是一个最简单的非抢占式算法。哪个进程先请求 CPU，就先为它分配 CPU。它使用一个简单的 FIFO 队列（先进先出）。
2. **短作业优先 (SJF）**，调度程序会从就绪队列中，挑选出**预计执行时间最短**的那个进程来执行。如果短作业一直到来，长作业将无法执行。
3. **轮转调度 (RR ）**，如果时间片用完，进程还没执行完，它就会被**强制中断**（抢占），并被放回就绪队列的**末尾**，等待下一轮。
4. **优先级调度 (Priority Scheduling)**， 为每个进程分配一个“优先级”（一个整数）。调度器总是选择**优先级最高**的进程来执行。

进程和线程的调度算法思想是一致的，但是实际操作对象不一致（调度对象）、上下文切换开销不一样（进程切换开销大，线程切换开销小）
