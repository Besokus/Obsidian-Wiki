# 🌟死锁，两个线程产生死锁的经典场景

当多个线程相互等待对方释放资源时，就可能导致死锁的发生。

通常情况下，死锁发生的四个必要条件是: 

**互斥条件**：资源一次只能被一个线程占用

**请求与保持条件**：线程已持有资源，还在请求其他资源

**不可剥夺条件**：资源不能被强制释放

**循环等待条件**：线程之间形成资源等待环



**经典场景**

有两把互斥锁：**Lock A**、**Lock B**

两个线程：**Thread 1**、**Thread 2**

1. **Thread 1**
   - 先获取 **Lock A**
   - 再尝试获取 **Lock B**
2. **Thread 2**
   - 先获取 **Lock B**
   - 再尝试获取 **Lock A**

如果时序不巧，就会出现：

- Thread 1 持有 A，等待 B
- Thread 2 持有 B，等待 A

👉 **双方互相等待，永远无法继续执行 —— 死锁**



**如何避免死锁**：

1. 银行家算法：资源分配前模拟是否会导致死锁，只有在系统处于安全状态下才允许进行资源分配
2. 打破产生死锁的四个必要条件
3. **统一加锁顺序**
4. **避免使用嵌套锁**：尽量避免在一个锁内部再次申请其他锁资源，可使用 `std::lock` 对两个mutex一次性加锁
5. **避免长时间持有锁**：减小临界区以减少锁的持有时间
6. 优化资源分配策略：评估和优化资源的分配策略，避免资源竞争和瓶颈情况的发生。
