# 🌟什么是进程，什么是线程，两者之间有什么联系，协程相比于这俩有什么区别

**进程**是程序的**执行实例**，是**系统进行资源分配和调度的基本单位**，每个进程都有独立的内存空间和系统资源。

**线程**是进程内的**执行单元**，是操作系统调度的最小单位，多个线程**共享进程的资源**（堆、全局变量），但有独立栈和寄存器上下文；

**联系**：线程属于进程，是进程内并发**执行的最小单位**；一个进程可以包含多个线程。

> **区别：**
>
> 1. 进程之间相互独立，线程之间共享同一进程的内存空间。
> 2. 进程切换开销大，线程切换开销小。
> 3. 一个进程崩溃不会影响其他进程，但一个线程崩溃可能导致整个进程崩溃。
> 4. 多进程更稳定，多线程更高效。



**协程**：协程是一种**用户态的轻量级并发单元**，由用户程序或运行时调度，切换开销远小于线程；而进程和线程由操作系统内核调度，开销更大。



**协程适合高并发、大量上下文切换场景，特别适合 I/O 密集型场景。但协程本身不提供真正的并行能力，通常需要与多线程结合使用。**



**协程劣势与不足**：

1. 协作式调度风险（最核心）：协程**不会被强制抢占**，必须主动 `yield / await`，如果计算太久没有await，会**饿死同线程上的其它协程**
2. CPU 密集型任务无优势：协程运行在线程之上，无法利用多核，**真正并行仍依赖多线程**
3. 对阻塞调用非常敏感：协程中调用阻塞 I/O、阻塞锁，会**阻塞整个线程**（解决：使用非阻塞I/O，协程感知的调度器）
