# Concept Template Guide

## 什么时候用 `Concept Template.md`
适合偏工程型内容：

- 协议落地
- 架构模式
- 中间件机制
- 调优与排障
- 组件 / 工具 / 策略

判断标准：

```text
如果你最关心的是：
怎么工作？
什么时候用？
什么时候不用？
出了问题怎么排查？
代价是什么？
```

那就用工程模板。

## 什么时候用 `Concept - Theory Template.md`
适合偏理论型内容：

- 定义
- 模型
- 性质
- 定理直觉
- 边界条件
- 反例
- 概念谱系

判断标准：

```text
如果你最关心的是：
它到底是怎么定义的？
基于什么假设？
保证什么性质？
为什么成立？
什么时候不成立？
```

那就用理论模板。

## 如果一个主题既有理论又有工程
优先按“这篇笔记的主问题”选模板：

- 主问题是“它是什么、为什么成立” → 理论模板
- 主问题是“怎么用、怎么实现、怎么排障” → 工程模板

必要时拆成两篇：

- 一篇理论笔记：定义 / 假设 / 性质 / 边界
- 一篇工程笔记：机制 / 选型 / 代价 / 排障

## 推荐实践
- 不要为了模板完整硬填每一节。
- 理论笔记优先写清“定义、假设、边界、反例”。
- 工程笔记优先写清“场景、机制、代价、故障、取舍”。
- 对重要概念补一个“易混概念”小节，复盘价值最高。
