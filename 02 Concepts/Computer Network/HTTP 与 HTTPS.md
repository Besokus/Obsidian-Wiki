# HTTP 与 HTTPS

## HTTP 请求方式

| 方法 | 含义 | 幂等 | 安全 |
|---|---|---|---|
| GET | 获取资源 | ✓ | ✓ |
| POST | 提交/创建 | ✗ | ✗ |
| PUT | 整体更新 | ✓ | ✗ |
| PATCH | 局部更新 | ✗ | ✗ |
| DELETE | 删除资源 | ✓ | ✗ |
| HEAD | 获取响应头 | ✓ | ✓ |
| OPTIONS | 查询支持方法 | ✓ | ✓ |

> **安全方法**：不会修改资源；**幂等方法**：多次执行结果相同。

### GET vs POST

| | GET | POST |
|---|---|---|
| 数据位置 | URL 中 | 请求体中 |
| 长度限制 | 有（URL 长度） | 无 |
| 缓存 | 可缓存 | 不可缓存 |
| 安全性 | 低（URL 可见） | 较高 |

## HTTP 状态码

### 分类

| 分类 | 含义 |
|---|---|
| 1xx | 信息 |
| 2xx | 成功 |
| 3xx | 重定向 |
| 4xx | 客户端错误 |
| 5xx | 服务端错误 |

### 常见状态码

| 状态码 | 含义 |
|---|---|
| 200 | OK |
| 301 | 永久重定向（浏览器会缓存新地址） |
| 302 | 临时重定向（下次仍访问原地址） |
| 304 | 未修改（使用本地缓存） |
| 401 | 未登录或 Token 失效 |
| 403 | 已登录但无权限 |
| 502 | 网关收到后端错误响应 |
| 504 | 网关超时未收到响应 |

## HTTP 缓存

### 强制缓存

浏览器判断资源是否命中强缓存，如果命中，直接从内存读取，不与服务器通信。

相关头部：`Cache-Control`（max-age）、`Expires`。

### 协商缓存

客户端与服务器交互，通过资源标识判断是否变化。

相关头部：
- `ETag` / `If-None-Match`（资源指纹）
- `Last-Modified` / `If-Modified-Since`（最后修改时间）

304 状态码表示资源未修改，可使用本地缓存。

## HTTPS = HTTP + SSL/TLS

### 加密机制

```text
对称加密：保证数据传输安全（快）
非对称加密：保证密钥交换安全（慢）
数字证书：保证服务器身份可靠
```

### 为什么同时用两种加密？

非对称加密安全但慢，对称加密快但密钥交换不安全。  
先通过非对称加密安全传输对称密钥，之后用对称加密高效通信。

### 证书被伪造怎么办？

浏览器会验证证书链（CA 签名），若签名无效或已过期，直接中断连接并报警。
