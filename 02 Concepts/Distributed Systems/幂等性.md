# 幂等性（Idempotency）

## 一句话解释
幂等性指：**同一个操作被重复执行多次，最终效果与执行一次相同**。它是“重试、恢复重放、消息重复投递”能安全工作的前提。

---

# 1. 为什么分布式系统离不开幂等
分布式系统中这些都很常见：
```text
超时 => 重试
网络抖动 => 重试
服务宕机恢复 => 重放
MQ at-least-once => 重复消息
```
如果不幂等，会造成：
```text
重复扣款 / 重复发货 / 重复释放库存 / 重复积分
```

---

# 2. 常见实现方式（工程清单）

## 2.1 幂等 key（请求去重）
- 客户端/上游生成 `Idempotency-Key`
- 服务端记录 `key -> result/status`（幂等表/去重表）

## 2.2 唯一约束（天然去重）
- 以业务唯一键落库（订单号、支付单号、消息 id）
- 依赖数据库唯一索引保证不重复插入

## 2.3 状态机门禁（只允许合法推进）
用状态机保证“只能从某些状态推进到下一状态”，重复请求直接返回当前状态结果（参见 [[事务状态机]]）。

## 2.4 版本号 / CAS
对同一资源并发更新：用版本号/条件更新避免重复扣减/覆盖。

---

# 3. 幂等与“可重复读”的区别
- 幂等解决的是“重复执行是否安全”
- 一致性模型解决的是“并发读写对外语义”（参见 [[一致性模型]]）
