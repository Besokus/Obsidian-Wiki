# 迭代器

## 迭代器的作用

迭代器提供了一种统一的方法来遍历不同容器中的元素，而无需了解容器的内部结构。可以视为指向容器中某个位置的指针。

## 迭代器失效

迭代器失效指容器被修改后，旧的迭代器/指针/引用不再有效。

### 不同容器的失效情况

| 容器 | 底层结构 | 失效场景 | 安全操作 |
|---|---|---|---|
| `vector`/`string` | 连续数组 | 扩容、中间插入/删除（当前+后续迭代器失效） | `reserve` 预分配，用 `erase` 返回值 |
| `list` | 双向链表 | 仅被 `erase` 删除的失效 | 其他迭代器有效 |
| `map`/`set` | 红黑树 | 仅被 `erase` 删除的失效 | 其他迭代器有效 |
| `unordered_map` | 哈希表 | rehash 全部失效，`erase` 局部失效 | 避免频繁 rehash |

### 安全遍历写法

```cpp
// vector 删除时用 erase 返回值
std::vector<int> v{1,2,3,4};
for (auto it = v.begin(); it != v.end(); ) {
    if (*it % 2 == 0)
        it = v.erase(it);   // 用返回值继续遍历
    else
        ++it;
}
```

### 规避方法

- 提前 `reserve` 预分配容量（vector）
- 修改后重新获取迭代器
- 需要稳定迭代器时选用 `list`/`map`
