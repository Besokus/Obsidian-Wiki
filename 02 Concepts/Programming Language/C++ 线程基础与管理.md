# C++ 线程基础与管理

## 创建线程

使用 `std::thread` 创建线程，可传入函数、函数指针、lambda 等。

```cpp
std::thread t1(func);
std::thread t2([](){ /* ... */ });
```

### 生命周期管理

| 方法 | 行为 | 适用场景 |
|---|---|---|
| `join()` | 主线程等待子线程结束 | 需要等待结果 |
| `detach()` | 线程独立运行，由运行时托管 | 后台任务、日志写入 |

`detach()` 后的线程无法再获取其状态或结果，也不保证生命周期。

---

## 线程安全

线程安全指在多线程并发访问下，程序仍能得到正确结果，不会出现数据竞争或状态不一致。

关键保障手段：

- **互斥锁**：保护临界区
- **原子操作**：轻量级无锁操作
- **RAII 封装**：`lock_guard` / `unique_lock` 自动管理锁生命周期
- **线程局部存储**：`thread_local` 关键字

---

## 多线程会带来的问题

| 问题 | 描述 | 解决 |
|---|---|---|
| 数据竞争 | 多个线程同时操作共享数据，结果不符合预期 | 锁 / 原子操作 |
| 死锁 | 线程互相等待对方持有的资源 | 统一锁顺序 / `std::lock()` |
| 活锁 | 线程不阻塞但重复无效操作 | 随机等待 / 重试上限 |
| 饥饿 | 某些线程长期得不到 CPU 时间 | 公平调度 / 线程池 |
| 上下文切换 | 线程过多 CPU 花在切换上 | 控制线程数 / 线程池 |
| 执行顺序不可测 | 线程执行顺序不确定 | 同步机制协调 |

---

## 线程崩溃

C++ 中一个线程崩溃（触发无法处理的错误）**会导致整个进程退出**。

常见原因：内存错误、非法指令、未捕获的异常。

---

## 多线程调试

- **日志**：记录线程 ID、锁获取/释放、关键状态变化
- **最小化复现**：把复杂场景缩减成最小多线程 demo
- **检测工具**：ThreadSanitizer、Helgrind 等
- **检查锁顺序**：排查死锁的核心路径
