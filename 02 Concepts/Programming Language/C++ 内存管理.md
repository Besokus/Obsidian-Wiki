# C++ 内存管理

## 堆 vs 栈

| | 栈 | 堆 |
|---|---|---|
| 存储 | 局部变量、函数调用信息 | 动态分配的数据 |
| 生命周期 | 函数执行周期 | 程序员显式控制 |
| 分配速度 | 快（自动） | 慢（手动） |
| 容量 | 有限 | 大 |

---

## new / delete vs malloc / free

| | new/delete | malloc/free |
|---|---|---|
| 本质 | C++ 运算符 | C 标准库函数 |
| 内存大小 | 自动计算 | 显式指定 |
| 类型安全 | 是 | 返回 `void*` 需强转 |
| 构造函数 | 调用 | 不调用 |

---

## 智能指针（RAII）

RAII：资源获取即初始化，构造获取资源，析构释放资源。

### unique_ptr

独占所有权，禁止拷贝，支持移动语义。适合独占资源的场景。

### shared_ptr

共享所有权，通过**引用计数**管理。内部包含两个指针：指向资源 + 指向控制块（引用计数等）。

使用 `make_shared` 可将对象和控制块一起分配，减少内存分配次数。

### weak_ptr

解决 `shared_ptr` 的**循环引用**。不增加引用计数，通过 `lock()` 获取 `shared_ptr`。

### 智能指针 vs 裸指针

| | 裸指针 | 智能指针 |
|---|---|---|
| 内存管理 | 手动 | 自动（RAII） |
| 安全性 | 低（泄漏、悬空） | 高（异常安全） |
| 性能 | 高（无额外开销） | shared_ptr 有引用计数开销 |
| 适用 | 高性能/底层场景 | 一般应用开发 |

---

## 内存泄漏

常见原因：忘记 delete、指针被覆盖、异常导致释放未执行、智能指针循环引用。

解决方法：
1. `new`/`delete` 成对出现
2. RAII 机制
3. 优先用智能指针，循环引用用 `weak_ptr`

## 浅拷贝 vs 深拷贝

**浅拷贝**：拷贝指针值，两个对象共用同一块内存，析构时双重释放。  
**深拷贝**：拷贝指针指向的实际数据，为新对象分配独立内存。

C++ 默认拷贝构造函数和赋值运算符都是**浅拷贝**，需要手动管理资源的类应自定义深拷贝。

```cpp
MyString(const MyString& other) {
    data = new char[strlen(other.data) + 1];
    strcpy(data, other.data);  // 深拷贝
}
```
