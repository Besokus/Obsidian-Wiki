# STL 容器

## Vector 扩容机制

### size() vs capacity()

- `size()`：当前已存储的元素个数
- `capacity()`：不分配新内存下最多可保存的元素个数
- `capacity >= size` 始终成立

当 `capacity == size` 时，vector 触发扩容。

### 扩容策略对比

| 策略 | 机制 | 优点 | 缺点 |
|---|---|---|---|
| 固定扩容 | 每次加固定值（如 +20） | 空间利用率高 | 可能频繁扩容，时间复杂度高 |
| 加倍扩容 | 每次翻倍（如 ×2） | 扩容次数少，均摊 O(1) | 空间利用率低 |

实际采用 **空间换时间** 策略，多用加倍扩容。

### 扩容过程

```text
1. 分配一块更大的内存（翻倍）
2. 将原元素复制/移动到新内存
3. 释放旧内存
4. 插入新元素
```

扩容会让指向原容器的**所有迭代器失效**。

## push_back vs emplace_back

| | push_back | emplace_back |
|---|---|---|
| 方式 | 创建副本或移动 | 就地构造 |
| 效率 | 有临时对象开销 | 省去副本创建 |
| 建议 | — | 构造成本高时优先用 |

## map / set / multimap / multiset

均基于**红黑树**实现，元素自动排序。

| 容器 | 存储 | 键唯一 | 访问方式 |
|---|---|---|---|
| `map` | 键值对 `pair<Key, Value>` | ✓ | `operator[]` |
| `set` | 键（元素） | ✓ | 迭代器 |
| `multimap` | 键值对 | ✗ | 迭代器遍历 |
| `multiset` | 键 | ✗ | 迭代器 |

## 哈希冲突

### 链地址法（Separate Chaining）

冲突时将元素插入对应桶的链表/红黑树尾部。

优点：负载因子可较大（如 0.75），可容纳任意多冲突键。

### 开放地址法（Open Addressing）

冲突时通过探测规则寻找空闲地址存储。

优点：数据存储连续，缓存命中率高。
