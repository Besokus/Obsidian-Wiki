# 线程池

## 为什么需要线程池

频繁创建和销毁线程的开销很大。线程池通过复用已创建的线程，减少线程创建/销毁的开销，并控制并发数量。

## 核心参数

| 参数 | 作用 |
|---|---|
| corePoolSize | 核心线程数，一直存活 |
| maximumPoolSize | 最大线程数 |
| keepAliveTime | 非核心线程空闲存活时间 |
| workQueue | 任务队列，等待执行的任务 |
| RejectedExecutionHandler | 拒绝策略（队列满且线程达上限） |

## 工作流程

```text
提交任务
  ↓
核心线程未满 -> 创建核心线程执行
  ↓
核心线程已满 -> 放入工作队列
  ↓
队列已满 -> 创建非核心线程执行
  ↓
线程达上限 -> 执行拒绝策略
```

## 线程数配置

### CPU 密集型（N + 1）

任务主要消耗 CPU（如大量计算、排序）。线程数设为 CPU 核心数 + 1（+1 是为了防止缺页中断等导致 CPU 空闲）。

### I/O 密集型（2N）

任务大部分时间在等待 I/O（网络、文件读写）。线程数可设为 2 × CPU 核心数。

## 拒绝策略

| 策略 | 行为 |
|---|---|
| AbortPolicy | 直接抛异常 |
| CallerRunsPolicy | 调用者线程自己执行 |
| DiscardPolicy | 直接丢弃 |
| DiscardOldestPolicy | 丢弃队列中最旧的任务 |
