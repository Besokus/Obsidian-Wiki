# 死锁

## 死锁的四个必要条件

1. **互斥条件**：资源一次只能被一个线程占用
2. **请求与保持条件**：线程已持有资源，还在请求其他资源
3. **不可剥夺条件**：资源不能被强制释放
4. **循环等待条件**：线程之间形成资源等待环

## 经典场景

```text
有两把锁：Lock A, Lock B

Thread 1: 获取 A -> 请求 B
Thread 2: 获取 B -> 请求 A

如果时序不巧：
  Thread 1 持有 A，等待 B
  Thread 2 持有 B，等待 A
  -> 双方互相等待，死锁
```

## 如何避免死锁

1. **统一加锁顺序**：所有线程按相同顺序获取锁
2. **避免嵌套锁**：使用 `std::lock` 一次性加多个锁
3. **避免长时间持有锁**：减小临界区
4. **银行家算法**：分配前模拟，安全才分配
5. **超时机制**：获取锁时设置超时，超时则释放已持有的锁
6. **使用无锁数据结构**：CAS 等原子操作替代锁
