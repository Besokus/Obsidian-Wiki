# 虚拟内存

## 什么是虚拟内存

虚拟内存是操作系统提供的一种内存管理机制，通过给每个进程分配独立的虚拟地址空间，实现内存隔离和扩展。

每个虚拟地址通过**页表**映射到实际的物理内存页。当访问未加载的页时会触发缺页中断，从磁盘加载数据。

## 为什么需要虚拟内存

1. **进程地址空间隔离**：每个进程有独立虚拟地址空间，提高安全性和稳定性
2. **内存扩展**：将硬盘空间作为临时内存使用，使程序看起来拥有比实际物理内存更大的空间
3. **按需加载**：只加载当前需要的页面，提高内存利用率
4. **内存保护**：支持只读、可执行等权限控制
5. **内存映射文件**：将文件 I/O 变成内存访问，效率更高

## 它回答的核心问题
- 它试图解释：为什么每个进程都像“拥有自己的连续内存”。
- 它试图约束：哪些地址可访问、哪些页面必须触发缺页。
- 它试图区分：虚拟地址空间和物理内存不是一回事。

## 重点部分详解
- 机制细节：虚拟地址先经过页表映射，再落到物理页；没有映射的页会触发缺页中断。
- 常见故障：页表过大、缺页过多、工作集超过物理内存时会引发明显性能抖动。
- 典型代价：地址翻译和页面置换都不是免费，内存看似“更大”，实际是用换页和管理成本换来的。
- 实战判断：程序突然变慢时，要考虑是否发生了大量缺页、抖动或内存回收。

## 页面置换算法

当物理内存不足时，需要将不常用的页面换出到磁盘：

| 算法 | 策略 |
|---|---|
| 最佳置换 OPT | 淘汰未来最远使用的（理想，不可实现） |
| FIFO | 淘汰最早进入的页面 |
| LRU | 淘汰最近最久未使用的 |
| Clock（NRU） | 近似 LRU 的环形扫描算法 |
| 改进型 Clock | 考虑修改位的 Clock 算法 |

## 和理论的关系
- 依赖的理论概念：地址空间、映射、局部性原理
- 关键假设：程序访问具有时间局部性和空间局部性。
- 理论边界对工程的影响：虚拟内存的“扩展感”来自按需加载，不是物理内存真的变多了。
