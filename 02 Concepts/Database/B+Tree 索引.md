# B+Tree 索引

## 一句话解释
B+Tree 是 MySQL 最常用的索引结构，适合磁盘场景下的范围查询、排序和有序扫描。它通过多路树降低树高，从而减少磁盘 IO 次数。

## 它解决什么问题
- 二叉树或红黑树在大数据量下层级深，磁盘 IO 次数多。
- Hash 索引适合等值查询，但不适合范围查询和排序。
- 数据库需要一种既能定位单点数据，也能高效范围扫描的结构。

## 核心机制
```text
1. 非叶子节点主要存储键和指针。
2. 数据集中出现在叶子节点。
3. 叶子节点按顺序连接，方便范围查询。
```

## 开发者视角
- 范围查询、排序和最左前缀匹配都和 B+Tree 的有序性有关。
- 主键越大，索引页可容纳的 key 越少，树可能更高。
- 随机主键可能导致页分裂，影响写入性能。

## 为什么 InnoDB 常用 B+Tree
- 相比二叉树，树高更低，磁盘 IO 更少。
- 相比 B-Tree，非叶子节点不保存完整数据，一页能放更多 key。
- 相比 Hash，支持范围查询和排序。
