# ACID

## 一句话解释
ACID 是事务可靠性的四个目标：**原子性、一致性、隔离性、持久性**。它描述的是数据库事务应该对外呈现出的行为。

## 它解决什么问题
- 原子性：防止操作只执行一半。
- 一致性：防止事务结束后违反业务约束。
- 隔离性：防止并发事务互相干扰。
- 持久性：防止事务提交后因崩溃丢失。

## 核心机制
```text
1. 原子性依赖 Undo Log 回滚。
2. 持久性依赖 Redo Log 恢复。
3. 隔离性依赖锁和 MVCC。
4. 一致性依赖数据库机制和业务约束共同保证。
```

## 开发者视角
- 不要把“一致性”只理解成数据库自动保证，业务约束也要通过代码、约束、事务边界共同维护。
- 隔离性越强，并发能力通常越低。
- 事务不是免费的，长事务会增加锁等待和版本保留成本。
