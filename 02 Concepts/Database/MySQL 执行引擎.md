# MySQL 执行引擎

## InnoDB（默认）

InnoDB 是 MySQL 默认的事务性存储引擎。

### 核心特点

- **支持事务**：完整 ACID 支持，通过 redo log 和 undo log 实现原子性和持久性
- **行级锁**：最大程度减少并发写入时的锁冲突
- **MVCC 多版本并发控制**：读操作通常不需要加锁，提高并发读性能
- **聚簇索引**：数据和主键索引存储在同一棵 B+Tree，主键查询效率高
- **崩溃恢复**：通过 redo log 在系统崩溃后恢复数据
- **外键约束**：维护参照完整性
- **缓冲池**：缓存数据和索引，提升读写性能

### 适用场景

高并发、事务支持、数据完整性要求高的应用。

## MyISAM

### 核心特点

- **非事务性**：不支持 ACID 事务，不支持外键
- **表级锁**：写操作锁定整个表，高并发写入时性能瓶颈明显
- **读取速度快**：数据和索引分开存储，结构简单
- **不支持崩溃恢复**：系统崩溃后可能丢失数据

### 适用场景

读多写少、对数据一致性要求不高的场景（如只读报表）。

## 对比

| 特性 | InnoDB | MyISAM |
|---|---|---|
| 事务 | 支持 | 不支持 |
| 锁粒度 | 行级 | 表级 |
| MVCC | 支持 | 不支持 |
| 外键 | 支持 | 不支持 |
| 崩溃恢复 | 支持 | 不支持 |
| 索引结构 | 聚簇索引 | 非聚簇索引 |
| 适用 | 高并发事务 | 只读/简单查询 |
