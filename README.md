## 你会在这里看到什么

- **Concept**：一个概念一篇（定义 → 解决什么问题 → 核心机制 → 工程判断 → 我的理解）
- **Problem**：工程问题诊断手册（现象 → 根因 → 定位路径 → 解决方案）
- **Source**：外部资料阅读笔记（核心观点 → 用处 → 与已有知识的连接）
- **Project**：实践记录（目标 → 设计 → 实现 → 观察结论）
- **MOC**：领域索引页（学习路径 + 概念地图）

## 写作原则

- **问题驱动**：先回答“它解决什么问题”，再补定义与机制
- **以自己的话复述**：不做资料搬运
- **链接优先**：用 `[[]]` 把真正相关的概念串起来
- **渐进拆解**：大主题拆成可复用的原子笔记

## 使用方式

### 在 Obsidian 里阅读（推荐）

1. 用 Obsidian 打开仓库根目录
2. 从 `01 MOC/` 的任意入口开始
3. `Ctrl+P` 搜索笔记名 / 用图谱或反向链接导航

### 在 GitHub 上阅读

- 该仓库大量使用 Obsidian 的 wikilink 语法（`[[...]]`），GitHub 不会自动解析成可点击链接；在 GitHub 阅读建议用目录 + 仓库搜索定位文件名。
