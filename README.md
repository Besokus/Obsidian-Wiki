# Besokus 的个人技术知识库

基于 **Obsidian** 构建的后端开发知识库，使用 Zettelkasten 和 MOC（Map of Content）方法组织知识。

## 目录结构

| 目录 | 用途 |
|---|---|
| `00 Inbox/` | 临时笔记草稿，定期清理归类 |
| `01 MOC/` | 领域索引页（Map of Content），梳理学习路径和概念关联 |
| `02 Concepts/` | 原子概念笔记，一个概念一篇 |
| `03 Sources/` | 外部资料的阅读笔记（博客、书籍、课程、论文） |
| `04 Problems/` | 工程问题诊断手册（故障、反模式、排查路径） |
| `05 Projects/` | 实验和实践项目记录 |
| `90 Templates/` | 笔记模板（Concept / Problem / Source / Project / MOC） |
| `99 Attachments/` | 图片、PDF 等附件 |
| `后端面试/` | 后端面试题整理 |

## 笔记类型

- **Concept** — 一个技术概念的深度拆解（定义 → 解决什么问题 → 核心机制 → 工程判断 → 我的理解）
- **Problem** — 一个工程问题的诊断手册（现象 → 根因 → 定位路径 → 解决方案）
- **Source** — 一份外部资料的阅读笔记（核心观点 → 对开发者的用处 → 与已有知识的连接）
- **Project** — 一个实践项目的完整记录（目标 → 设计方案 → 实现 → 观察结论）
- **MOC** — 领域索引页，组织学习路径和概念地图

## 核心原则

- **问题驱动** — 先回答"它解决什么问题"，再写定义
- **必须写自己的理解** — 不摘抄资料
- **双向链接** — 每篇笔记通过 `[[]]` 与真正相关的笔记连接
- **渐进拆解** — 大型笔记逐步拆成原子笔记

## 覆盖领域

- 分布式系统（复制、一致性、共识、故障模型）
- 数据库（MySQL/InnoDB 原理、Redis、事务、索引）
- 计算机网络
- 操作系统
- C++
- 消息队列

## 使用方式

1. 用 Obsidian 打开本仓库根目录
2. 从 `01 MOC/` 进入任一领域索引页开始学习
3. 用 `Ctrl+P` 搜索笔记名
4. 点击 `[[]]` 双向链接在概念间导航
